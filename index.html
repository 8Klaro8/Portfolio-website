Brief Description of the app.

This app was 100% built by me. My goal with this was to gain experince thru creating something that I feel exciting and challenging, and also to have my own app as a portfolio project. Below you can see what the app is capable of.

Technologies used

API

I used the Fitness Calculator from Rapidapi

The app uses the burned calorie from activity API. It calculates the burned calorie based on the user's weight and the workout's duration. The weight is given when the user registrates and the duration is a must fill in field when the user adds a new workout.

JavaFX

First I used only Swing but I wanted to give it a better look.

Whit JavaFx I had some challanges to implement but overall it was not difficult to learn it it is it very similar to Swing. What was challenging is to migrate the entire project to JavaFx, since the project was close to finished when I started to implement it. Definitely worth it.

Maven

I came along with Maven and I realized, better to learn it.

Some time I came along with Maven, because I needed to add a package and I realized, that it will be much effective if I use Maven as my build tool. It was also a migration but it was less time consuming. Also Maven is a tool that worth using. Makes the devepoling process more effective.

PostreSQL

The app needed a Database to save users and all their details.

In postgres I came along with a few challanges and the biggest was JSON. Frist I intended to save data in JSON but I realized it is not the best practice so I changed to storee data in different formats.

Detailed description

Below I go a little bit more in dept how the app works.

First you find yourself on the login page where you are able to choose between login or register. If you register then you have to add a few details, including height and weight. The app uses the weight with an API and the height is there for future developing, maybe for an other API.

When the user registers then the password is hashed and salted, then it stores in PostgreSQL with te rest of the details. I managed Postgres with PgAdmin.

Once logged in the user is able to set his or her details on the settings page, also able to choose from several profile images and enter to the add workout page which allows the user to add new workout, edit or delete. The user can choose title, type, icon, and duration for the workout. Only title and type are editable afterwards.
